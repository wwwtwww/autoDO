# LarkAutoClock 待办事项 (TODO)

## 🎯 第一阶段：实用功能完善（高优先级）

### 1. 应用内打卡日志查看 (Log Viewer)
*   **背景**：极速打卡的结果目前保存在应用沙盒内部的 `clock_log.txt`，非 Root 设备难以直接读取。
*   **任务**：在 `MainActivity` 中增加“查看打卡记录”功能，支持将本地日志读取并在新页面或弹窗中按时间倒序展示。
*   **目的**：让用户直观地看到每次定时任务的执行时间及打卡是否成功。

### 2. 打卡时间开放配置 (Time Configuration)
*   **背景**：当前上下班的基准打卡时间和随机防风控偏移量（如 7:30~8:20、18:00~18:10）是硬编码在 `ClockScheduler.kt` 中的，修改需重新编译。
*   **任务**：在主界面新增时间配置选项（如：TimePicker 控件），使用 `SharedPreferences` 持久化保存用户的自定义时间，并在 `ClockScheduler` 中读取这些配置动态计算闹钟。
*   **目的**：适应不同用户的上下班时间需求，提升应用的通用性。

## 🛠 第二阶段：进阶打磨与稳定性提升（中优先级）

### 3. 失败重试与容错机制 (Retry Mechanism)
*   **背景**：网络波动、飞书后台卡死或系统卡顿等极端情况可能导致极速打卡失败。
*   **任务**：如果在 15 秒内未检测到打卡成功标志，程序自动尝试强杀飞书进程（利用 `ActivityManager` 或再次冷启动），等待 2~3 分钟后重新尝试拉起。
*   **目的**：进一步提高无人值守状态下的打卡成功率。

### 4. App 主界面美化与状态展示 (UI/UX Polish)
*   **背景**：目前界面仅作为开发调试用，按钮罗列较为简陋。
*   **任务**：
    *   重构 UI 设计，使其更具现代感（如卡片式设计）。
    *   加入状态面板：清晰展示“调度系统运行状态”、“今日节假日状态”、“下一次预计打卡时间”等核心信息。
*   **目的**：增强用户体验，让应用看起来像一个成熟的成品软件。
